# Ukrainian Idioms Extractor

Вебзастосунок для автоматичної екстракції фразеологізмів з тексту, що поєднує два взаємодоповнювальні підходи: словниковий та велику мовну модель. (Натисніть, аби перейти до [англомовної версії]((README.md)))

---

## Демо

Застосунок доступний на Hugging Face Spaces — без встановлення: [Ukrainian Idiom Extractor](https://huggingface.co/spaces/anastasiia-mozg/Ukrainian-Idiom-Extractor)

## Як це працює

Застосунок розбиває введений текст на речення та запускає два незалежні пайплайни видобування паралельно:

- **Словниковий метод** — порівнює речення зі списком фразеологізмів із цього [репозиторію](https://github.com/bohdana-ivakhnenko/ukrainian-idioms). Перед пошуком і текст, і записи словника лематизуються за допомогою української моделі spaCy, що дозволяє виявляти фразеологізми у різних відмінках.
- **Метод моделі (Gemini)** — надсилає речення до Google Gemini 2.5 Flash Lite, яка визначає фразеологізми на основі семантичного та контекстуального розуміння, без використання фіксованого словника.

Результати обох методів відображаються поруч, а також виділяються фразеологізми, знайдені одночасно обома методами.

## Можливості

- Обробка українських текстів довільного обсягу
- Виявлення фразеологізмів у різних відмінках
- Порівняння результатів словникового методу та LLM з визначенням збігів
- Інтерактивна стовпчикова діаграма з кількістю знайдених фразеологізмів
- Завантаження результатів у форматі CSV
- Зручний інтерфейс на Streamlit (вступ → введення тексту → результати)

## Встановлення

**1. Клонуйте репозиторій**

```bash
git clone <repo-url>
cd Ukrainian-Idioms-Project
```

**2. Встановіть залежності**

```bash
pip install -r code/requirements.txt
```

**3. Завантажте українську модель spaCy**

```bash
python -m spacy download uk_core_news_lg
```

**4. Додайте ключ Google API**

Створіть файл `secrets.env` у кореневій директорії проєкту (цей файл додано до `.gitignore`):

```
GOOGLE_API_KEY=ваш_ключ_тут
```

**5. Запустіть застосунок**

```bash
streamlit run code/main.py
```

## Як будувався словник

Клас `VocabExtractor` відповідає за попередню обробку вихідних даних словника. Починаючи з TSV-файлу з фразеологізмами та цитатами, він:

1. Розбирає нотацію у квадратних дужках для генерації варіантів поверхневих форм — наприклад, `[не]` породжує форми з цим словом і без нього, а `слово(синонім)` розгортається у кілька варіантів
2. Генерує додаткові варіанти шляхом перестановки порядку слів і заміни взаємозамінних сполучників та прийменників (`і/й/та`, `у/в`)
3. Лематизує всі варіанти за допомогою spaCy для подальшого зіставлення з лематизованим вхідним текстом

Результат зберігається у `dicts/idiom_variants_dict.json`.
